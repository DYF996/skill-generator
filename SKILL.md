# Skill Generator

## 自然语言命令
- `直接执行 需求` 或 `做 需求`：直接执行，不输出计划
- `先计划 需求` 或 `计划 需求`：先输出步骤清单，确认后执行
- `生成技能 需求` 或 `创建技能 需求`：生成完整 Skill 包（SKILL.md + 可选资源）

## 自动判断（无命令时）
- 含"做/帮我/执行/处理" → 直接执行模式
- 含"步骤/计划/怎么" → 先计划模式
- 含"生成/创建 Skill/技能" → 生成技能模式

## 简化模式
- 用户输入"简化模式"后，AI 用自然语言总结步骤（不展示技术细节）
- 输入"详细模式"或"显示步骤"恢复技术清单
- 模式状态在对话中记忆，每次回复前检查

## 执行规则
### 先计划模式
1. 输出步骤清单（每步一行，简要说明）
2. 询问："确认执行？(回复"确认"或提出修改)"
3. 用户确认后逐步执行，关键节点给出进度
4. 某步失败则停止并提示："第N步失败，请手动处理后重新执行"

### 直接执行模式
- 直接执行，失败则停止并提示

### 生成技能模式
- 输出目录结构（如 `name/SKILL.md`、可选的 `references/`、`scripts/`、`assets/`）
- SKILL.md 必须包含：
 ```yaml
 ---
 name: 技能名
 description: 一句话描述触发场景和功能
 ---
 # 标题
 ## 适用场景
 ## 执行步骤
 ## 约束条件
 ## 示例（至少一个）
 ```

## 重复性任务检测与 Skill 生成建议

在每次任务执行完成后（直接执行模式或先计划模式确认执行后），AI 应自动分析本次任务是否可能为重复性任务。

### 判断依据（满足任一即触发询问）
- 用户输入含周期词：每天/每周/每月/每年/每日/周报/日报/月报/例行/定时
- 用户输入含固定路径模式：`*.csv`、`~/data/`、`/logs/`、`report_*.xlsx`
- 用户输入含"经常"、"每次"、"总是"、"重复"等词
- 操作步骤数 ≥ 3 且涉及外部数据源
- 用户明确要求"以后还要做"或类似表述

### 询问模板

---
🔁 检测到该任务可能具有重复性（原因：<具体原因>）。
是否需要为这个任务创建一个 Skill？下次您只需一句话就能自动完成，可节省约 70% 的 token 消耗。
回复"是"立即生成 Skill，回复"否"忽略。
---

### 防止过度询问
- 同一任务特征（输入的前 50 个字符 + 主要操作）在同一对话中只询问一次。
- 用户连续拒绝 2 次后，本次对话不再主动询问任何任务的 Skill 生成。
- 用户发送"不再询问生成 Skill"后，本对话内永久关闭主动询问。
- 用户使用 `生成技能` 命令时，不触发此询问。
- 用户明确在任务描述中说"不要生成 Skill"时，跳过询问。

### 询问后的处理
- 用户回复"是"：自动进入 `生成技能` 流程，基于本次任务的具体步骤生成 SKILL.md，并询问用户指定 Skill 名称（如不指定则自动命名）。
- 用户回复"否"：记录拒绝次数，继续对话。
