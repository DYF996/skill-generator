# Skill Generator

> ⚠️ **重要说明**：当前 AI 无法自动保存文件到你的电脑，也无法跨对话记忆。本技能生成的 Skill 需要你**手动复制内容并保存**到本地（例如 `~/.claude/skills/` 或自定义目录）。新对话时，请将 Skill 内容粘贴给 AI 或描述任务名称即可复用。

## 自然语言命令
- `直接执行 需求` 或 `做 需求`：直接执行，不输出计划
- `先计划 需求` 或 `计划 需求`：先输出步骤清单，确认后执行
- `生成技能 需求` 或 `创建技能 需求`：生成完整 Skill（单文件文本块，可一键复制）

## 自动判断（无命令时）
- 含"做/帮我/执行/处理" → 直接执行模式
- 含"步骤/计划/怎么" → 先计划模式
- 含"生成/创建 Skill/技能" → 生成技能模式

## 简化模式
- 用户输入"简化模式"后，AI 用自然语言总结步骤（不展示技术细节）
- 输入"详细模式"或"显示步骤"恢复技术清单
- 模式状态在对话中记忆，每次回复前检查
- **注意**：生成技能模式始终使用详细模式（不受简化/详细模式影响）

## 执行规则
### 先计划模式
1. 输出步骤清单（每步一行，简要说明）
2. 询问："确认执行？(回复"确认"或提出修改)"
3. 用户确认后逐步执行，关键节点给出进度
4. 某步失败则停止并提示："第N步失败，请手动处理后重新执行"
5. 失败后询问："是否重试？(是/否) 或 '修改需求' 重新开始"

### 直接执行模式
- 直接执行，失败则停止并提示，然后询问是否重试或修改需求

### 生成技能模式（适配当前场景）
当用户要求生成 Skill 或同意生成后，AI 应输出**单个完整文本块**，包含：

1. **Skill 基本信息**（名称、描述）
2. **完整 SKILL.md 内容**（YAML frontmatter + 正文）
3. **可选的脚本/参考内容**（如果任务需要脚本，以代码块形式附在同一个消息中）
4. **保存和使用的说明**（告诉用户如何保存到本地、如何在下次对话中使用）

**输出格式示例**（用户只需复制整个代码块）：

```markdown
--- 复制以下内容到文件：my_skill/SKILL.md ---
---
name: my-skill
description: 描述...
---
# 标题
## 适用场景
...
## 执行步骤
...
## 约束条件
...
## 示例
...
--- 可选：如果需要脚本，另存为 scripts/run.py ---
```python
# 脚本内容
```

--- 使用说明 ---

1. 创建文件夹 my_skill/
2. 保存 SKILL.md 和脚本
3. 新对话中说："使用 my-skill 技能 处理 XXX"

```

## 重复性任务检测与 Skill 生成建议

在每次任务执行完成后（直接执行模式或先计划模式确认执行后），AI 应自动分析本次任务是否可能为重复性任务。

### 判断依据（满足任一即触发询问）
- 用户输入含明确周期词：每天/每周/每月/每年/每日/周报/日报/月报/例行/定时
- 用户输入含固定路径模式：`*.csv`、`~/data/`、`/logs/`、`report_*.xlsx`
- 用户输入含"经常"、"每次"、"总是"、"重复"、"以后还要做"等词
- 用户明确要求"以后还要做"

**不再使用"步骤数≥3"条件**，以减少误判。

### 询问模板

---
🔁 检测到该任务可能具有重复性（原因：<具体原因>）。
是否需要为这个任务创建一个 Skill？下次您只需一句话就能自动完成，可显著降低 token 消耗（通常 50%–80%）。
回复"是"立即生成 Skill，回复"否"忽略。
---

### 防止过度询问
- 同一任务特征（输入的前 50 个字符 + 主要操作）在同一对话中只询问一次。
- 用户连续拒绝 2 次后，本次对话不再主动询问任何任务的 Skill 生成。
- 用户发送"不再询问生成 Skill"后，本对话内永久关闭主动询问。
- 用户使用 `生成技能` 命令时，不触发此询问。
- 用户明确在任务描述中说"不要生成 Skill"时，跳过询问。

### 询问后的处理
- 用户回复"是"：自动进入 `生成技能` 流程，基于本次任务的具体步骤生成 Skill 文本块。
- 用户回复"否"：记录拒绝次数，继续对话。

## 增强功能：立即试用刚生成的 Skill
在生成 Skill 后，AI 应主动询问：
> 是否在当前对话中试用这个 Skill？您只需提供一组测试输入（或使用刚才的任务数据），我会模拟执行 Skill 中的步骤，验证效果。
- 用户回复"是"：AI 使用生成的 SKILL.md 中的"执行步骤"模拟运行一次（不要求真实文件操作）。
- 用户回复"否"：结束流程，提供保存提示。

## 跨对话使用指南（每次生成 Skill 后附带）

📁 如何保存并使用这个 Skill：

1. 创建文件夹，例如 my_skill/
2. 将上面的 SKILL.md 内容保存到该文件夹（文件名必须为 SKILL.md）
3. 如果有脚本，保存到 scripts/ 子文件夹
4. 在新对话中，告诉 AI："请加载 ~/my_skill/ 中的 Skill" 或者直接说"使用 my-skill 技能做 XXX"
 （注意：AI 不会自动加载，你需要手动粘贴 SKILL.md 内容到新对话，或使用平台支持的 Skill 加载机制）

## 适用场景
- 用户想自动执行一次性或重复性任务，并希望 AI 自动检测重复性、建议生成 Skill。
- 用户需要将一组操作封装成可复用的 Skill，并能手动保存和跨对话使用。

## 执行步骤
1. 解析用户输入，判断模式（直接执行/先计划/生成技能）。
2. 根据模式执行任务（执行步骤、生成 SKILL.md 等）。
3. 任务完成后，依据判断条件分析是否为重复性任务，若是则询问是否生成 Skill。
4. 根据用户反馈进入生成技能流程，输出可复制的文本块。
5. 询问是否立即试用生成的 Skill（可选）。
6. 提供保存和跨对话使用说明。

## 约束条件
- AI 无法自动保存文件，需要用户手动复制和保存。
- AI 无法跨对话记忆 Skill，新对话需要用户重新提供 Skill 内容或使用平台支持的 Skill 加载机制。
- 生成的脚本可能需要用户根据实际环境调整（路径、权限、依赖等）。
- 重复性检测依赖关键词，可能漏判或误判。
