# 交互式问题求解协议

> 本文件是 `skill-generator` 的交互式问题求解模块。包含完整的执行流程、风险分级机制、检查点与回滚规范、简化/详细双模式规范。

---

## 触发条件

当用户提出一个具体问题需要解决（而非明确要求生成 Skill），且用户选择了 **A（直接执行）** 或 **B（先计划再确认）** 时，触发本流程。

---

## 步骤 1：自主分析问题

拆解用户需求，填写以下分析框架（记忆在上下文）：

```
**目标**：用户想达成什么？
**输入**：用户提供什么数据/材料？（若缺失，优先推断；推断不出则反问）
**输出**：用户期望得到什么结果？
**关键步骤**：解决这个问题需要哪几步？（控制在 ≤ 8 步）
**可能的坑点**：哪些地方容易出错或需要特别注意？
**工具/方法**：用什么工具/代码来完成？
**预估风险等级**：低 / 中 / 高（见"风险分级"章节）
```

> **信息不足处理原则**：优先从上下文推断；无法推断时必须反问，禁止凭空假设。反问示例：
> - "这个文件的格式是 CSV 还是 Excel？"
> - "您说的'整理'是指按月份汇总，还是按产品分类？"
> - "有没有需要特别注意的列或关键词？"

---

## 步骤 1.5：可行性自检与风险评估

在生成步骤计划前，**先做可行性自检**，避免计划不可执行：

### 检查项

| 检查项 | 说明 |
|--------|------|
| **文件访问** | 文件路径是否存在？权限是否足够？ |
| **依赖可用性** | 所需 Python 库/工具是否已安装？（pandas、matplotlib 等） |
| **API 密钥** | 所需 API（如 Tavily、TTS）是否已在环境变量中配置？ |
| **数据格式** | 所需处理的文件格式是否已知且可解析？ |
| **外部资源** | 所需 URL/API 是否可达？是否需要网络？ |
| **输出目录** | 目标输出路径是否可写？磁盘空间是否足够？ |

### 自检结果处理

**全部通过** → 继续生成步骤计划，注明"✅ 环境检查通过"

**发现问题** → 在步骤计划中标注：

```markdown
### ⚠️ 环境提醒
- ❌ <缺失项>：<说明> → 请先<解决方案>，否则相关步骤可能失败
- ⚠️ <风险项>：<说明> → 已准备替代方案：<替代方案>
```

> 自检不能替代执行阶段的真实错误捕获——有些问题（如 API 限流）只在执行时才暴露。

---

## 步骤 2：风险分级

在步骤计划中，为每个步骤标注风险等级：

| 等级 | 定义 | 处理方式 |
|------|------|----------|
| 🟢 **低风险** | 只读操作、无副作用、不修改外部状态 | 自动执行，无需确认 |
| 🟡 **中风险** | 写入文件、调用外部 API、可能产生费用 | 需审核（除非用户选"跳过后续审核"） |
| 🔴 **高风险** | 删除文件、执行系统命令、修改配置、不可逆操作 | 必须逐条审核，即使选择了"跳过后续审核"也需单独确认 |

### 低风险操作示例（自动执行）
- 读取本地文件
- 生成分析报告
- 执行纯计算
- 调用已配置的 API（Tavily、DeepSeek）

### 中风险操作示例（需审核）
- 创建或写入新文件
- 调用可能收费的 API
- 下载外部资源

### 高风险操作示例（必须确认）
- 删除文件或目录
- 执行系统命令（`rm -rf`、`del`）
- 修改系统配置
- 发送外部消息（邮件、微信）

---

## 步骤 2.5：检查点创建与日志记录

### 检查点创建（执行前）

在开始执行前，将当前上下文快照存入内存（不落盘，不影响用户数据）：

```
检查点 #1 @ <时间戳>
- 用户需求：<摘要>
- 已确认的步骤计划：<步骤列表>
- 当前模式：<简化/详细>
- 跳过审核：<是/否>
- 上下文摘要：<关键变量和数据引用>
```

### 执行日志（每步执行后）

记录格式：

```
[@ 步骤 N | <时间戳>]
✅ 步骤 "<名称>" 完成
  产出：<关键产出摘要>
  耗时：<Xs>

[@ 步骤 N+1 | <时间戳>]
⏳ 步骤 "<名称>" 进行中...
```

### 副作用记录

每步若产生文件写入、API 调用等副作用，同步记录：

```
副作用：
  - 创建：<file path>
  - 调用：<API endpoint>
  - 修改：<config file>
```

## 步骤 3：编写步骤计划

将分析结果转化为 Markdown 步骤计划：

```markdown
## 📋 任务执行计划

**目标**：<一句话描述>
**风险等级**：🟢 低 / 🟡 中 / 🔴 高（见下方说明）
**预估时间**：<X 分钟>

### ✅ 环境检查
- <已通过项>
- ❌ <缺失项> → <解决方案>

### 执行步骤

1. 🟢 **<步骤名称>**：<做什么> → 使用 <工具/方法> [低风险 → 自动执行]
2. 🟡 **<步骤名称>**：<做什么> → 使用 <工具/方法> [中风险 → 需审核]
3. 🔴 **<步骤名称>**：<做什么> → 使用 <工具/方法> [高风险 → 必须确认]

### 预期输出
<描述最终产出的格式和内容>

### 回滚方案
- 若步骤 2 失败：回滚到检查点，从步骤 1 重新开始
- 若步骤 3 失败：跳过步骤 3，输出前序结果
```

### 跳过后续审核选项

在计划末尾加一行：

> 📌 **审核选项**：回复"**确认**"→ 开始执行（低风险步骤自动执行，中高风险逐条确认）；回复"**跳过**"→ 全部步骤按计划执行（含中高风险），无需进一步确认；回复"**取消**"→ 终止任务。

---

## 步骤 4：用户确认

等待用户回复，处理方式：

| 用户回复 | 处理 |
|----------|------|
| **确认、执行、开始、开始吧、可以** | 进入执行阶段 |
| **跳过、跳过后续审核** | 进入执行阶段，跳过所有审核直接执行 |
| **取消、算了、不用了** | 终止，回复："好的，任务已取消，随时有需要再说。" |
| **提出修改意见** | 调整计划，**重新提交审核**（可循环多轮） |
| **反问或闲聊** | 先回答问题，然后重新请求确认 |

---

## 步骤 5：执行并输出结果

### 执行原则

1. **按步骤顺序执行**，不跳步（除非用户明确同意跳过某步）
2. **每步完成后输出简报**：进度 X/Y，产出摘要
3. **遇到错误**：停步，说明原因，提供选项（回滚 / 跳过 / 终止）
4. **中高风险步骤前**：再次确认（如用户选"跳过"，本步自动执行）

### 简化模式输出（默认非技术用户）

- 用自然语言描述在做什么，不展示代码
- 进度用 Emoji 和简短句子：`⏳ 正在读取您的文件...` `✅ 读取完成，共 2800 行数据`
- 最终结果用清晰的文字 + 表格 + Emoji 展示，不展示原始代码

### 详细模式输出

- 展示每步使用的命令、代码片段、工具参数
- 记录完整执行日志
- 提供可复制粘贴的代码块

---

## 步骤 4.5：错误处理与回滚

### 错误分类

| 错误类型 | 示例 | 处理方式 |
|----------|------|----------|
| **环境错误** | 缺少依赖、文件不存在、权限不足 | 停止，提示用户解决后再试 |
| **数据错误** | 格式不匹配、空值过多、编码问题 | 尝试自动处理（填充空值、转换编码），失败则停步询问 |
| **API 错误** | 限流、超时、密钥无效 | 重试 1-2 次，失败则停步，提供替代方案 |
| **逻辑错误** | 步骤判断出错、输出不符合预期 | 回滚到上一步，检查点恢复 |

### 回滚操作

**触发条件**：用户说"回滚"、或某步骤失败且选择回滚

**回滚执行**：
1. 从检查点恢复上下文
2. 删除本步已产生的副作用文件
3. 从上一步继续，或询问用户是否重试当前步骤

**回滚选项**：
> ⚠️ 步骤 N 执行失败：<错误摘要>
>
> 请选择：
> - **回滚**：回到上一步检查点，删除本步产出，重新尝试
> - **跳过**：放弃本步，继续执行后续步骤（可能输出不完整）
> - **终止**：停止任务，保留已完成的产出

---

## 步骤 6：固化询问

全部完成后，询问：

> ✅ 任务完成！您是否需要将以上步骤保存为一个可复用的 Skill？回复"**是 + 名称**"（例如：是 sales-report），我来生成。

- **用户同意** → 跳转到 `skill_spec.md` 流程，本次步骤作为 SKILL.md 核心内容
- **用户拒绝** → 结束本次任务，存档上下文

## 简化模式 vs 详细模式规范

### 简化模式（默认）

**适用对象**：非技术背景用户
**输出风格**：
- 纯自然语言，不用代码块
- 用 Emoji 标注进度
- 最终结果：文字描述 + 表格 + 图表占位说明
- 遇到问题用人话解释

**示例**：
```
⏳ 正在读取您的 Excel 文件... ✅ 已加载，共 2800 行
⏳ 正在按月份汇总销售额... ✅ 完成！Q3 最高（¥38万）
⏳ 正在生成图表... ✅ 柱状图 + 饼图已生成
✅ 全部完成！
```

### 详细模式

**适用对象**：技术用户、开发者
**输出风格**：
- 展示完整代码块、命令、参数
- 记录每步执行日志
- 提供可复制的代码
- 说明可选参数和变体

## 回滚完整示例

### 执行到步骤 3 时出错

```
[@ 步骤 3 | 14:32:05]
⏳ 正在读取 API 数据...
❌ 步骤 3 执行失败
  错误：HTTP 429 — 请求过于频繁，API 限流
  已尝试：1 次重试（等待 5 秒），仍然失败

⚠️ 请选择：
- **回滚**：回到步骤 2 检查点，删除本步产出，重新尝试步骤 3
- **跳过**：放弃 API 数据，改用本地缓存数据继续（结果可能不完整）
- **终止**：停止任务，保留步骤 1-2 的产出
```

### 用户选择"回滚"

```
[@ 14:32:15]
↩️ 正在回滚到检查点 #2...
  ✅ 已删除步骤 3 产出文件（temp_api_response.json）
  ✅ 已恢复上下文状态
  ⏳ 重新执行步骤 3（等待 10 秒后重试 API）...
```
